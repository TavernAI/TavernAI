Der Schnellstart führt vom Öffnen von TavernAI 2 bis zur ersten Antwort der AI.

Nutze ihn nach der [Installation](/de/docs/installation/). Wir starten die App, verbinden einen Provider, fügen eine Karte hinzu und senden eine Nachricht.

## 1. Erste Nachricht

### 1.1 TavernAI starten

Öffne TavernAI aus dem Ordner, den du entpackt hast.

![TavernAI launch placeholder](/img/docs/1.png)

### 1.2 Provider verbinden

Öffne die Provider-Einstellungen und wähle das AI-Modell, das du verwenden willst.

![Provider setup placeholder](/img/docs/2.png)


### 1.3 Chat erstellen

Drücke den **Chat**-Button des Charakters, um einen neuen Chat zu starten.

![Create chat placeholder](/img/docs/3.png)

### 1.4 Nachricht schreiben

Jetzt schreiben wir die erste Nachricht.

![Add card placeholder](/img/docs/4.png)

### 1.5 Antwort erhalten

Jetzt erhalten wir die erste AI-Antwort.

![First message placeholder](/img/docs/5.png)

## 2. Karten hinzufügen

Zu diesem Zeitpunkt sprechen wir noch nicht als character card. Fügen wir eine Karte für uns hinzu, damit der Chat einen Teilnehmer auf unserer Seite hat.

### 2.1 Kartenpanel öffnen

Öffne das Panel zum Hinzufügen von Karten zum Chat.

![First message placeholder](/img/docs/6.png)

### 2.2 Charakter hinzufügen

Wähle die Karte, die du für dich verwenden willst.

![First message placeholder](/img/docs/7.png)

### 2.3 Karte konfigurieren

Die Karte befindet sich jetzt im Chat.

![First message placeholder](/img/docs/8.png)

Aktuell ist die AI so eingestellt, dass sie für beide Karten spricht. Weisen wir Scout als unsere Karte zu und lassen Marika als Karte auf AI-Seite.

So sieht Scout aus, wenn die AI für diese Karte spricht:

![First message placeholder](/img/docs/9.png)

Und so sieht Scout aus, nachdem die Karte zu unserer Karte geworden ist:

![First message placeholder](/img/docs/10.png)

Die controls der Karte bedeuten:

`AI` — Karten mit diesem Flag nehmen an AI-generated messages teil.

`U` — Karten mit diesem Flag nehmen an deinen Nachrichten teil.

`Ctx` — wenn aktiviert, wird der Prompt Manager der Karte zum generation context hinzugefügt.

`Role selector` — steuert die Chat Completion role für Nachrichten dieser Karte, damit das Modell weiß, ob sie als Nachrichten der Nutzerseite oder der AI-Seite behandelt werden sollen.

### 2.4 Nachricht als Karte senden

Jetzt senden wir eine Nachricht als Scout. Marika reagiert anders, weil der Chat nun einen Charakter auf unserer Seite hat.

![First message placeholder](/img/docs/11.png)

## 3. Rollen tauschen

Jetzt erstellen wir einen neuen Chat und kehren die Karteneinstellungen um.

![First message placeholder](/img/docs/12.png)

Diesmal spielen wir Marika, die Besitzerin der Nachtbäckerei, und empfangen Scout als Gast.

![First message placeholder](/img/docs/13.png)
