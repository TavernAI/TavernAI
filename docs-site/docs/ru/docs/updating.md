TavernAI можно обновлять из интерфейса. Ручные способы нужны только тогда, когда интерфейсное обновление недоступно или нужно перенести установку вручную.

## Обновление из интерфейса

1. Откройте TavernAI.
2. Перейдите в `Settings` -> `Updates`.
3. Установите обновление.

Updater скачивает подходящую сборку, проверяет ее, подготавливает в `update_cache` и заменяет файлы приложения после выхода из TavernAI. После успешного обновления `update_cache` очищается. Локальные данные остаются на месте: `user_data`, `logs`, `models` и `config.yaml`.

Дайте TavernAI завершить все startup migration перед повторным закрытием приложения.

## Ручное обновление

Используйте этот способ, если обновление из интерфейса нельзя применить или если нужно заменить сборку вручную.

1. Полностью закройте TavernAI.
2. Сделайте резервную копию старой папки `user_data`.
3. Скачайте новую сборку со страницы [Download](/download/).
4. Распакуйте новую сборку в новую папку.
5. Скопируйте `user_data` из старой папки в новую.
6. Запустите новую сборку.

## Обновление VPS

Если TavernAI был установлен через VPS script и нужно обновить его из терминала, запускайте updater из папки приложения:

```bash
cd ~/TavernAI
./updater/update.sh
```

Если вы установили TavernAI в другое место, запустите `updater/update.sh` из этой папки.

## Что сохранять

Сохраняйте всю папку `user_data`.

В ней находятся база данных, чаты, импортированные файлы, изображения и другое локальное состояние TavernAI.

## Перенос TavernAI

TavernAI можно переносить папкой. Чтобы перенести его в другую папку или на другой диск:

1. Закройте TavernAI.
2. Скопируйте всю папку приложения в новое место.
3. Запустите TavernAI из нового места.
4. Проверьте, что чаты и файлы на месте.

## Не делайте так

- Не удаляйте `user_data` во время обновления.
- Не используйте одну и ту же live-папку `user_data` из двух разных версий TavernAI.
- Не используйте старую сборку как способ отката для `user_data`, которая уже открывалась в новой версии.
